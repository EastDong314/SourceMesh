# SourceMesh - AI Agent 开发指南

## 项目概述

**SourceMesh** 是一个基于 Playwright CLI 的数据收集中心，为下游系统（如 TradingP 和 SignalReactor）提供可重用的市场情报数据。

### 核心目标
- 集中化管理网页爬取任务
- 规范化提取的数据格式
- 支持多种数据输出目标（数据库、Redis、其他系统）
- 提供稳定的 API 接口供其他项目调用

## 开发规范

### 1. 数据收集器 (Collectors)

每个数据源对应一个 Collector，遵循以下结构：

```typescript
interface Collector {
  name: string;           // 唯一标识
  source: string;         // 数据源 URL
  schedule: string;      // Cron 表达式
  extract(): Promise<Data>;
  transform(raw: RawData): NormalizedData;
}
```

### 2. 数据存储

#### PostgreSQL 表结构
```sql
CREATE TABLE collected_data (
  id UUID PRIMARY KEY,
  source VARCHAR(100) NOT NULL,
  type VARCHAR(50) NOT NULL,
  data JSONB NOT NULL,
  collected_at TIMESTAMP NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_source ON collected_data(source);
CREATE INDEX idx_collected_at ON collected_data(collected_at);
```

#### Redis 数据结构
- **List**: `queue:pending` - 待处理数据队列
- **Hash**: `data:{source}:latest` - 各数据源最新数据
- **Sorted Set**: `data:{source}:timeline` - 时间线数据

### 3. API 接口

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/sources` | GET | 列出所有数据源 |
| `/api/data/:source` | GET | 获取指定源的数据 |
| `/api/trigger/:source` | POST | 手动触发采集 |
| `/api/health` | GET | 健康检查 |

## 常用命令

```bash
# 安装依赖
npm install

# 开发模式
npm run dev

# 运行采集器
npm run collect          # 运行所有
npm run collect -- --source=xxx  # 运行指定源

# 数据库迁移
npm run db:migrate

# 构建
npm run build

# 测试
npm run test
```

## 注意事项

1. **反爬策略**: 遵守 robots.txt，控制请求频率
2. **数据合规**: 确保数据采集符合相关法律法规
3. **敏感信息**: 使用环境变量管理 API Keys 和数据库凭证
4. **错误处理**: 所有异步操作必须有完善的异常捕获
